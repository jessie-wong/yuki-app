## 开始搭建

1. 搭建 next 基础框架

```bash
npx create-next-app@latest yukilab --typescript
npx next build
npm run start
```

2. markdown 配置
   https://nextjs.org/docs/app/guides/mdx

   2.1 安装依赖——md 解析

   npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
   运行时动态加载 md
   访问路径：/doc/welcome

   运行后发现 md 文档没有样式，继续探索如何渲染 md

   - 使用@tailwindcss/typography 的样式

     ```ts
     import typography from "@tailwindcss/typography";
     /** @type {import('tailwindcss').Config} \*/
     const config = {
       content: [
         "./app/**/_.{js,ts,jsx,tsx,mdx}",
         "./components/**/_.{js,ts,jsx,tsx,mdx}",
         "./content/*_/_.{md,mdx}",
       ],
       theme: {
         extend: {},
       },
       plugins: [typography],
     };
     export default config;
     ```

     ```css
     @import "tailwindcss";
     @import "@tailwindcss/typography";
     ```

   ````
    - 在 MDX 文件中使用 GFM 语法（比如表格、任务列表、删除线等）；
    - 代码高亮

    ```ts
    // 引入 hightlight 样式
     import remarkGfm from "remark-gfm";
     import rehypeHighlight from "rehype-highlight";
     import "highlight.js/styles/github-dark.css";

    const { content } = await compileMDX({
      source,
      options: {
        parseFrontmatter: true,
        mdxOptions: {
          // 在 MDX 文件中使用 GFM 语法（比如表格、任务列表、删除线等）；
          remarkPlugins: [remarkGfm],
          rehypePlugins: [rehypeHighlight],
        },
      },
      // 动态导入时，需要使用 getMDXComponents 获取组件
      components: getMDXComponents({}),
    });
   ````

- 自定义样式：mdx-components.tsx
